Universal gotchas live in `cowork-github-replit-process/reference/06-known-gotchas.md`. This file captures CherryAI-specific traps.

## Razor / dotnet gotchas

### `@()` inside C# interpolated strings → compile error

`$"{count} attachment@(count == 1 ? \"\" : \"s\")"` → parser error "Unexpected (".

**Rule:** Pre-compute the pluralized word in an `@{}` block above, reference the local in the interpolation.

```cshtml
@{
    var attachWord = count == 1 ? "attachment" : "attachments";
    var attachDelta = $"{count} {attachWord} on file";
}
```

**Hit in:** PR #116d.6 (Asset Detail attachments KPI).

### `@{}` inside `if/else {}` body → compile error

Inside an `if/else { }` body, you're already in code context. Don't double-wrap with `@{}`.

```cshtml
@* WRONG *@
@if (cond) {
    @{ var x = 1; }
}

@* RIGHT *@
@if (cond) {
    var x = 1;
}
```

**Hit in:** PR #116d.10 (Reliability page initial commit).

### `class="prefix@variable"` triggers Razor email detection

```cshtml
@* WRONG — rendered literally *@
<div class="ds-skel-grid@extra">

@* RIGHT *@
<div class="ds-skel-grid@(extra)">
```

**Hit in:** PR #116d.1b.1 (SkeletonLoader fix).

### `_ScreenHeader` propagates `headerVd`, NOT outer ViewData, to KpisPartial

When a page uses `_ScreenHeader` + `KpisPartial`, the inner partial reads `headerVd`'s ViewData, not the page's outer ViewData. This is because `_ScreenHeader.cshtml` calls `Html.PartialAsync(kpisPartial)` without explicit ViewData — so the inner partial inherits whatever ViewData is in scope at the call site (which is `headerVd`).

**Pattern that works:**
```csharp
// In page-level cshtml — set keys BEFORE constructing headerVd
ViewData["TotalEntries"] = totalEntries;
ViewData["TotalLines"] = totalLines;

var headerVd = new ViewDataDictionary(ViewData) {
    ["HeaderTitle"] = "...",
    ["KpisPartial"] = "_MyKpis",
    // Also set keys on headerVd directly (belt + suspenders)
    ["TotalEntries"] = totalEntries,
    ["TotalLines"] = totalLines
};
```

OR pass explicit `KpisViewData`:
```csharp
var headerVd = new ViewDataDictionary(ViewData) {
    ["KpisPartial"] = "_MyKpis",
    ["KpisViewData"] = new ViewDataDictionary(ViewData) {
        ["TotalEntries"] = totalEntries
    }
};
```

**Hit in:** PR #116d.12.1 (Journals KPI ViewData ordering fix).

### EF lazy-loaded navigation collections in KPI partials = 0

If a partial reads `entity.RelatedItems.Count` but renders BEFORE EF has materialized the collection, count is 0. The same property read later (e.g. in the page body table) returns N. Same symptom as the ViewData propagation issue but different cause.

**Symptom:** KPI shows 0, table below shows N rows.

**Fix:** Pre-compute the values in page-level cshtml (forces EF materialization), pass via explicit `KpisViewData`:

```csharp
var preLineCount = po.Lines?.Count ?? 0;     // force-load
var preReleaseCount = po.Lines?.Sum(l => l.Releases?.Count ?? 0) ?? 0;

var headerVd = new ViewDataDictionary(ViewData) {
    ["KpisViewData"] = new ViewDataDictionary(ViewData) {
        ["PurchaseOrder"] = po,
        ["LineCount"] = preLineCount,
        ["ReleaseCount"] = preReleaseCount
    }
};
```

**Hit in:** PR #116d.16.1 (PO Detail KPI lazy-load fix).

## CherryAI design system

### Tone semantics — match meaning, not just "looks pretty"

- `success` (green) — confirmed-good states ONLY (balanced books, complete tasks)
- `warning` (amber) — needs attention (pending approvals, partial states)
- `danger` (red) — error / out-of-balance / overdue
- `info` (blue) — descriptive / neutral data
- `brand` (cherry-red gradient) — primary content area / hero CTAs
- `neutral` (gray) — filter cards / structural containers

Don't over-use `brand`. Reserved for the most important content area on each page.

### Tone-flipping deltas — required, not optional

KPITile delta text should change tone/direction based on the value's meaning:
- "X items overdue" → danger / down
- "Y items balanced" → success / up
- "Z items pending" → warning / down
- "N total" → info / flat (no judgment)

If a delta is a static label ("descriptive metric"), use `flat` direction + `muted` tone. Don't fake-flip a static value.

### Em-dash for zero-value cells in tables

In Journal Lines, PO Lines, etc., cells with $0 value should render as `—` not `$0.00`. Accountants read em-dashes as "no value here" — cleaner scanning.

```cshtml
<td>@(amount > 0m ? amount.ToString("C2") : "—")</td>
```

## Replit cache

### `bin/obj` must be cleared BEFORE Agent restart for any `.cshtml` change

`*.Views.dll` is AOT-compiled at build time and cached. Web Server "restart" doesn't recompile views.

```bash
rm -rf bin obj
# THEN trigger Agent restart
```

This is in the universal skill but worth re-stating because it bites every Razor PR.

## Sites + companies + tenancy

CherryAI EAM is multi-company. Most queries filter by `_tenantContext.VisibleCompanyIds`. If a page seems empty when it shouldn't be, check:
1. Is the user logged in?
2. Does the user have access to the company that owns the data?
3. Is the data scoped to `CompanyId.HasValue && visibleIds.Contains(...)` correctly?

The "Acting as" badge in the screen header shows the current role context.

## Naming collisions

`Abs.FixedAssets.Models` is crowded. Always GREP before adding a new enum or class:

```bash
grep -r "enum YourNewEnum" Models/
```

**Previously bit on:** `UnitOfMeasure`, `WorkOrderCategory`.

## Test posture quick reference

- **Layer 1** (DB schema): `tests/db-validate-*.sh` — bash scripts against Replit's Postgres
- **Layer 2** (integration): `tests/integration-*.sh` — service-level scenarios
- **Layer 3** (HTTP smoke): `tests/route-smoke-*.sh` — every route returns 200/302
- **Layer 4** (browser): Playwright specs in `Tests/` — some are age-flaky
- **xUnit** (`Abs.FixedAssets.Tests`): 9 skipped specs, intentional — rewrite backlog item

CI runs Layer 1-3 + xUnit. Layer 4 runs locally only.
