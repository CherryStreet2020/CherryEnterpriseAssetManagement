# Side Navigation Audit — 2026-05-25

**Audit purpose:** Dean: *"The side menu is a disaster and many of the links are old or point to old or non-existing pages."* This memo feeds **Sprint 13.6 PR #2 (NavRegistry cleanup)**.

**Audit scope:**

- `Services/Navigation/NavRegistry.cs` (single source for sidebar items outside the Control Center spine — Sprint 13.5 PR #4)
- `Services/Navigation/ControlCenterRegistry.cs` (Control Center spine — ADR-017 §D2)
- `Pages/Shared/_NavSidebar.cshtml`, `_NavControlCenters.cshtml`, `_NavQuickActions.cshtml`
- `Program.cs` `AddPageRoute` aliases
- All `Pages/**/*.cshtml` `@page` directives and `ViewData["Title"]` values

**Verification rules:**

1. A nav `Route` like `/Foo/Bar` resolves if `Pages/Foo/Bar.cshtml` exists OR `Pages/Foo/Bar/Index.cshtml` exists OR a `@page "/Foo/Bar"` directive points to a different file OR `Program.cs` has an `AddPageRoute(...)` alias.
2. `@page "{id:int}"` (parameterised) without an optional marker (`{id:int?}`) means navigating to the bare URL returns 404 — flagged as broken.
3. Pages that exist but are unreachable from sidebar / CC spine / quick actions are flagged as **orphans**.
4. Where the NavRegistry `Title` and the destination page's `ViewData["Title"]` drift meaningfully, flagged as **label mismatch**.
