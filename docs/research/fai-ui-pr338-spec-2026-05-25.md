## Why this PR exists

The 2026-05-25 demo-readiness audit identified FAI UI as the **#1 highest-leverage single
feature** for the aerospace audit story (Rolls-Royce / Goodrich / Raytheon all require
AS9102 sign-off on first articles). PR #1.75 shipped the schema-only foundation
(FaiReport / FaiCharacteristic / FaiProductAccountability) but the UI was never built —
biggest false-positive in the codebase.

## Scope (tight, demo-driven)

**IN:**
- `IFaiService` with 6 methods (List / GetById / Create / RecordCharacteristic /
  RecordProductAccountability / SignOff).
- 3 Razor pages: `/Quality/Fai/Index`, `/Quality/Fai/Create`, `/Quality/Fai/Detail/{id}`.
- Per-tenant monotonic FAI numbering ("FAI-2026-00001") via simple LAST+1 lookup with
  tenant scope (no NumberSequence integration — that service is WorkOrder-specific).
- Wire-up from `/CustomerProjects/Details/{id}` — a "First Article Inspections" section
  with a list of FAIs linked to this project + "New FAI" button.
- Cockpit-light pages (use `_ScreenHeader` pattern from /Admin/AssetImport; reserve full
  Cockpit primitives for the future Quality Control Center).
- AS9102 vocabulary (Form 1 / Form 2 / Form 3, ballooned drawing references, conformance
  enum, MRB disposition link).

**OUT (later PRs):**

## Service contract

```csharp
public interface IFaiService
{
    Task<IReadOnlyList<FaiReport>> ListAsync(int companyId, long? customerProjectId, CancellationToken ct);
    Task<FaiReport?> GetByIdAsync(long id, CancellationToken ct);
    Task<FaiReport> CreateAsync(FaiCreateRequest req, int userId, string? username, CancellationToken ct);
    Task<FaiCharacteristic> RecordCharacteristicAsync(long faiReportId, FaiCharacteristic row, int userId, string? username, CancellationToken ct);
    Task<FaiProductAccountability> RecordProductAccountabilityAsync(long faiReportId, FaiProductAccountability row, int userId, string? username, CancellationToken ct);
    Task<FaiReport> SignOffAsync(long faiReportId, int approverUserId, string? approverName, CancellationToken ct);
    Task<FaiReport> SubmitAsync(long faiReportId, int submitterUserId, string? submitterName, CancellationToken ct);
}

public sealed record FaiCreateRequest(
    int CompanyId, int? TenantId, int ItemId, int? CustomerProjectId, int? CustomerId,
    string PartNumberSnapshot, string? PartNameSnapshot, string? DrawingNumberSnapshot,
    string? DrawingRevSnapshot, FaiType Type, FaiPartType PartType, FaiReason Reason,
    string? ReasonText);
```

## FAI Numbering

Simple per-tenant LAST+1: `SELECT MAX(SUBSTRING(FaiNumber FROM 'FAI-{YEAR}-(\d+)$')) + 1`.
Wrap in a transaction with table lock. Format: `FAI-{YYYY}-{counter:00000}` (5 digits =
99,999 per year — plenty for any tenant).

## Status workflow

UI offers buttons matching the FaiStatus enum:
- Draft → "Submit" → Submitted
- Submitted → "Approve" → Approved (requires ApprovedById != SubmittedById per AS9102 Rev C)
- Submitted → "Reject" → Rejected
- Approved → "Mark Conditional" (waiver applied) → Conditional
- Any non-Voided → "Void" (only Admin role)

Status-regression Postgres trigger (PR #1.75) enforces Draft→Voided is illegal.
