## 1. The scenario in one sentence

A 10-level precision-machining bracket assembly demo project at PWH MANUFACTURING CANADA, with one parent assembly + 9 sub-assembly production orders, one routing per order with 3-5 operations, ONE subcontract operation at a generic finishing vendor, backward-scheduled dates, pre-stamped cost values, last 3 operations at a secondary site for the cross-plant beat.

**Hero question for Shadi (COO persona):** "Show me where this part is right now, what it costs so far, and what's running it through the back of the shop."

**Re-running the seeder is safe** — natural-key checks find existing rows and skip the bucket.

All postings use SYSTEM template GL accounts (already on dev, `CompanyId IS NULL`); facility scoping rides on `AccountingKey` segmentation per Lock 15:

| AcctNum | Name | Use |
|---|---|---|
| 1310 | WIP Inventory — Production | Dr on accumulation |
| 5200 | Production Labor Expense | Cr on labor |
| 5210 | Production Overhead | Cr on OH application |
| 5340 | Overhead Applied | Contra-OH |
| 5500 | WIP-to-FG Clearing | Future FG receipt |
| 2160 | Accrued Labor | AP credit on labor accruals |

Per JE: stamps `AccountingKey { CompanyId=2, LocationId=<MAIN|NORTH>, ... }` for site-scoped reporting. The single GL account + multi-dimensional AccountingKey is the correct architecture (Lock 15) — proliferating WIP-MAIN / WIP-NORTH accounts would be the anti-pattern.
